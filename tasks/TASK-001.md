# TASK-001: 初始遊戲實現 (Initial Game Implementation)

## 任務目標
實現《神兵猜拳》的功能完整單人練習版本，包含所有核心機制和美術風格。

## 核心需求

### 1. 遊戲機制
- 實現基礎猜拳規則（石頭、布、剪刀）
- 三大派系系統：劍、盾、槍
- 戰前佈局：派系選擇 + 強化卡選擇
- HP系統（10點）+ 護甲機制
- 5回合限制

### 2. 派系能力
- **劍派系**：強化卡造成傷害時額外+1傷害（總計3點）
- **盾派系**：平手時獲得1層護甲
- **槍派系**：上回合受傷時本回合傷害+1

### 3. UI/UX要求
- 爐石戰記風格設計
- 移動優先響應式設計
- 厚重木石質感UI框架
- 強化卡金色光環動畫

### 4. 技術實現
- HTML5 + CSS3 + JavaScript
- 模組化架構（core_logic.js, single_player.js）
- AI對手系統
- 完整的遊戲流程

## 交付物
- 完整可玩的單人版本
- 所有核心機制正常運作
- 響應式設計在移動端和桌面端都完美運行
- AI對手提供合理挑戰

## 驗收標準
- [ ] 遊戲可以正常啟動並顯示主畫面
- [ ] 戰前佈局功能完整（派系+強化卡選擇）
- [ ] 5回合對戰流程順暢
- [ ] 三大派系能力正確觸發
- [ ] AI對手行為合理
- [ ] 移動端和桌面端響應式設計完美
- [ ] 遊戲結束後正確顯示結果
